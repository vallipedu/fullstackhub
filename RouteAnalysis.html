Route Analysis: Chennai to Delhi

Problem Statement10

Objective

Design and develop a web application to analyze a travel route from Chennai to Delhi. The system must process a dataset of connecting cities and their distances to calculate the total journey time.

Functional Requirements

  • Calculate Total Journey Time: The system must calculate and display the total journey time in hours, assuming a constant travel speed of 100 km/hour.

Technical Specifications

  • Data Structure: The route data must be implemented as a JavaScript array of objects. Each object will represent a connecting city and must include properties for name, description, and the distance from the previous city.
  • Sample Records: 
    
[
    {
        name: "Chennai",
        description: "Starting point of the journey.",
        distance: 0
    },
    {
        name: "Hyderabad",
        description: "A major tech hub and connecting city.",
        distance: 630
    },
    {
        name: "Nagpur",
        description: "Central Indian city, often a major stop on this route.",
        distance: 510
    },
    {
        name: "Delhi",
        description: "The final destination.",
        distance: 1090
    }
]
  • Programming Language: JavaScript for all application logic. The solution must make extensive use of modern JavaScript syntax.
  • User Interface: A simple interface built with HTML and a minimal amount of CSS to display the route information and the calculated travel time.

Testing Requirements

  • Total Journey Time Test:
    • Test Case 1.1: Given a complete route dataset, the application must correctly calculate the total travel time.
    • Expected Result: A single numeric value representing the total journey time in hours is displayed accurately.
