Student Enrollment Status

Problem Statement7

Objective

Design and develop a web application for a university's student management system. The system must process a dataset of student records to display their enrollment status for the Full-Stack AI course.

Functional Requirements

  • Display Enrollment Status: For a given student, the system must display whether they are "Enrolled" or "Not Enrolled" based on their record.

Technical Specifications

  • Data Structure: The student data must be implemented as a JavaScript array of objects. Each object will represent a single student and must include properties for id, name, class, gpa, and enrollmentStatus.
  • Sample Records: 
    
[
    {
        id: 101,
        name: "Vijay Kumar",
        class: "III B.Tech CSE",
        gpa: 8.5,
        enrollmentStatus: true
    },
    {
        id: 102,
        name: "Saranya Devi",
        class: "III B.Tech IT",
        gpa: 7.9,
        enrollmentStatus: false
    },
    {
        id: 103,
        name: "Arjun Reddy",
        class: "III B.Tech CSE",
        gpa: 9.1,
        enrollmentStatus: true
    }
]
  • Programming Language: JavaScript for all application logic. The solution must make extensive use of modern JavaScript syntax.
  • User Interface: A simple interface built with HTML and a minimal amount of CSS to display the student data and the results.

Testing Requirements

  • Enrollment Status Test:
    • Test Case 1.1: Given a student with enrollmentStatus: true, the application must correctly display "Enrolled".
    • Test Case 1.2: Given a student with enrollmentStatus: false, the application must correctly display "Not Enrolled".
    • Expected Result: A clear message is displayed for each student, indicating their enrollment status.
