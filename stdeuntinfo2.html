Student Information System

Problem Statement 6

Objective

Design and develop a web application for a university's student management system. The system must process a dataset of student records and determine their pass or fail status based on their marks.

Functional Requirements

  • Display Pass/Fail Status: For a given student, the system must display whether they have passed or failed.
  • Pass Condition: A student passes if their average mark across all subjects is greater than 50. Otherwise, they fail.

Technical Specifications

  • Data Structure: The student data must be implemented as a JavaScript array of objects. Each object will represent a single student and must include properties for name, id, and their subject marks.
  • Sample Records: 
    
[
    {
        id: 101,
        name: "Priya Sharma",
        English: 85, Maths1: 75, Maths2: 90, Chemistry: 65, Physics: 80
    },
    {
        id: 102,
        name: "Rohan Singh",
        English: 45, Maths1: 55, Maths2: 40, Chemistry: 48, Physics: 52
    },
    {
        id: 103,
        name: "Anjali Gupta",
        English: 95, Maths1: 88, Maths2: 92, Chemistry: 76, Physics: 85
    }
]
  • Programming Language: JavaScript for all application logic. The solution must make extensive use of modern JavaScript syntax.
  • User Interface: A simple interface built with HTML and a minimal amount of CSS to display the student data and the results.

Testing Requirements

  • Pass/Fail Test:
    • Test Case 1.1: Given a student with an average score greater than 50, the application must correctly display "Pass".
    • Test Case 1.2: Given a student with an average score of 50 or less, the application must correctly display "Fail".
    • Expected Result: A clear message is displayed for each student, indicating either "Pass" or "Fail".
