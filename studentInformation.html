Student Information System

Problem Statement5

Objective

Design and develop a functional web application for a university's student management system. The system should process a dataset of student information and provide a simple interface to display and search for students.

Functional Requirements

  • Display Student List: The system must display a list of all students in a clean, tabular format.
  • Search Functionality: The system must provide a search bar to check whether a particular student exists in the dataset.

Technical Specifications

  • Data Structure: The student data must be implemented as a JavaScript array of objects. Each object will represent a single student and must include properties for id, name, and branch.
  • Sample Records: 
    
[
    { id: 101, name: "Alice", branch: "CSE" },
    { id: 102, name: "Bob", branch: "ECE" },
    { id: 103, name: "Charlie", branch: "ME" }
]
  • Programming Language: JavaScript for all application logic. The solution must make extensive use of modern JavaScript syntax.
  • User Interface: A simple interface built with HTML and a minimal amount of CSS to display the student data and the search functionality.

Testing Requirements

  • Student Search Test:
    • Test Case 1.1: Given a complete student dataset, searching for a student who exists must correctly return a positive result.
    • Test Case 1.2: Given a complete student dataset, searching for a student who does not exist must correctly return a negative result.
    • Expected Result: A clear message is displayed indicating whether the student was found or not.
