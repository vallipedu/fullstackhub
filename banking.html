Banking Transaction System

Problem Statement

Objective

Design and develop a web application that manages a customer's bank account. The system must process a series of transactions to calculate and display the final account balance.

Functional Requirements

  • Calculate Final Balance: The system must calculate and display the final account balance after a series of deposits and withdrawals.

Technical Specifications

  • Data Structure: The transaction data must be implemented as a JavaScript array of objects. Each object will represent a single transaction and must include properties for id, type (e.g., "deposit" or "withdrawal"), and amount.
  • Sample Records: 
    
[
    {
        id: "T001",
        type: "deposit",
        amount: 5000.00
    },
    {
        id: "T002",
        type: "withdrawal",
        amount: 1500.00
    },
    {
        id: "T003",
        type: "deposit",
        amount: 2500.00
    }
]
  • Programming Language: JavaScript for all application logic. The solution must make extensive use of modern JavaScript syntax.
  • User Interface: A simple interface built with HTML and a minimal amount of CSS to display the transaction history and the final balance.

Testing Requirements

  • Final Balance Test:
    • Test Case 1.1: Given a complete transaction dataset, the application must correctly calculate the final account balance.
    • Expected Result: A single numeric value representing the final balance is displayed accurately.
